Quickstart

Trois étapes : récupérez une clé API en beta, pointez votre SDK vers https://api.infera.eu/v1, appelez un modèle.

Beta: l'accès est sur invitation. Rejoindre la waitlist · les clés sont émises sous 48h.

Authentication

Bearer token dans l'en-tête Authorization. Les clés sont préfixées infera_sk_, révocables à chaud, rotation recommandée tous les 90 jours.

HEADERAuthorization: Bearer infera_sk_•••

Niveaux

Sélectionnez le tier via l'en-tête X-Infera-Tier. Par défaut : public. Trusted et HDS requièrent un contrat.

  • public — dev, tests, best-effort.
  • trusted — production B2B, SLA 99.5%.
  • hds — données de santé & financières régulées.

Endpoints

POST /v1/chat/completions Chat & completions, support streaming
POST /v1/embeddings Embeddings (768 / 1024 / 4096 dims)
GET /v1/models Catalogue des modèles disponibles

Streaming

SSE standard OpenAI. Ajoutez stream: true. Les chunks sont identiques (delta format).

Function calling

Support complet sur modèles Llama 3.3, Mistral Large, Qwen 2.5. Interface tools / tool_choice identique à OpenAI.

Errors

Codes HTTP standard. Payload JSON identique OpenAI (error.message, error.type, error.code). Codes spécifiques Infera préfixés infera_.
